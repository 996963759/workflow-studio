# 安全与边界

## 已实现

- 本地账号注册 / 登录。
- Bearer Token 认证。
- 工作流、运行历史、知识库按用户隔离。
- 工具节点默认只允许请求 `localhost`、`127.0.0.1`、`::1`。
- 知识库上传限制为 Markdown / TXT。
- 单个知识文档限制为 1MB。
- 模型 API Key 通过环境变量配置，不返回给前端。

## 当前边界

- 密码使用 PBKDF2 哈希保存，但还没有密码重置流程。
- Token 当前没有过期时间和刷新机制。
- 当前没有团队空间、角色权限和管理员后台。
- 当前没有 HTTPS 配置，需要部署层提供。
- 当前没有审计日志和操作回放。
- 当前知识库不是向量检索。

## 生产化建议

- 使用 PostgreSQL。
- 增加 token 过期、刷新和撤销策略。
- 引入用户级模型密钥加密保存。
- 增加团队、角色、项目空间权限模型。
- 部署时使用 HTTPS 和反向代理。
- 增加结构化审计日志。
