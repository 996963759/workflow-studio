# 工作流编辑器需求与迭代计划

## 目标

把当前前端 Demo 推进成一个可实际试用的类 Dify 工作流编辑器。当前阶段重点不是接入真实模型，而是先让用户能够搭建、保存、恢复、导入、导出，并按连线顺序运行前端工作流逻辑。

## 目标用户

- 想用可视化方式编排 AI 应用流程的产品经理、开发者和运营人员
- 想快速验证 AI 工作流结构，但暂时不想先搭后端和数据库的个人开发者

## MVP 范围

当前阶段要支持：

- 可视化节点画布
- 节点新增、拖拽、连线、删除
- 节点配置编辑
- 输出变量展示
- 模拟运行预览
- 本地保存到浏览器
- 刷新页面自动恢复
- JSON 导入和导出
- 重置为示例工作流
- 本地多工作流管理
- 工作流列表搜索、排序、归档和恢复
- 工作流名称编辑
- 后端连接状态展示
- 当前工作流同步到后端
- 从后端加载工作流列表
- 后端运行历史记录
- 前端查看后端运行历史
- 运行历史按工作流筛选和清理
- 运行历史搜索、状态筛选和整次结果复制
- 服务端工作流结构校验
- 后端保存、更新和运行工作流前阻止严重结构问题
- 前端优先展示后端校验结果，后端离线时回退到本地校验
- 大模型节点支持后端真实 DeepSeek / OpenAI 调用，未配置密钥时回退模拟输出
- 前端展示后端模型配置状态，不暴露 API Key 内容
- 模型调用失败时展示结构化错误原因，并回退模拟输出
- 知识检索节点支持本地 Markdown / TXT 文档检索
- 工具节点支持后端真实本机 HTTP 请求
- 后端运行按连线拓扑顺序执行，不依赖节点位置
- 按节点连线计算执行顺序
- 运行时传递变量上下文
- 支持 `{{变量名}}` 模板替换
- 支持简单“包含 / contains”条件判断
- 条件节点支持真 / 假两个出口，并按判断结果路由
- 运行日志展示节点输入、输出和写入变量
- 运行日志支持复制单节点输入、输出、错误原因和整次运行结果
- 工作流检查面板展示严重问题和提醒
- 校验问题支持点击定位到对应节点
- 严重问题会阻止运行
- 有问题的节点会在画布上显示警告状态
- 节点配置面板按节点类型展示专属字段
- 节点配置支持字段级校验和默认值恢复
- 运行输入可以手动编辑或套用示例
- 变量可以一键插入到提示词、检索语句、工具请求地址、请求头、请求体、判断值和输出模板
- 条件节点通过表单配置变量、判断方式和判断值

暂不支持：

- 用户登录和权限
- 知识库上传和向量索引
- 外网工具白名单管理
- 团队协作和版本历史

## 节点类型

- 用户输入：收集用户请求和变量
- 知识检索：从本地 Markdown / TXT 文档中检索相关片段
- 大模型：根据系统提示词、用户提示词模板和上下文变量模拟生成草稿
- 工具调用：调用本机 HTTP API，或在未配置请求地址时返回模拟结果
- 条件分支：根据表单化条件配置进行路由
- 最终回答：组装并返回最终结果

## 数据结构

工作流定义包含：

- `name`：工作流名称
- `version`：定义版本
- `nodes`：节点 ID、位置和节点配置
- `edges`：连线 ID、起点和终点
- `updatedAt`：本地保存时间

## 当前实现方案

当前阶段包含前端工作台和 FastAPI 后端雏形：

1. 使用 React Flow 管理画布、节点和连线。
2. 使用 React state 管理当前工作流。
3. 使用 `localStorage` 保存最新工作流定义。
4. 使用隐藏文件输入导入 JSON。
5. 使用 Blob 下载导出 JSON。
6. 保存、导入、重置后清理运行预览，避免用户误读旧结果。
7. 运行时使用拓扑排序计算节点执行顺序。
8. 执行过程中维护变量上下文，并将节点输出写入 `outputKey`。
9. 使用 `{{变量名}}` 语法在提示词、检索语句、工具请求地址、请求头、请求体、输出模板和判断值中替换变量。
10. 条件节点支持结构化配置：判断变量、判断方式、判断值。
11. 每次渲染时校验工作流结构，并将问题映射到节点警告态。
12. 点击运行时，如果存在严重问题，直接展示校验错误并阻止执行。
13. 运行输入由右侧运行预览面板提供，输入节点会把它写入输出变量。
14. 配置面板提供变量插入按钮，避免用户手动输入模板变量。
15. 使用 `workflow-studio.workflows` 保存多个本地工作流。
16. 使用 `workflow-studio.active-workflow-id` 保存当前选中的工作流。
17. 导入 JSON 时创建新的工作流，而不是覆盖当前工作流。
18. 旧版 `workflow-studio.current-workflow` 会自动迁移成工作流列表中的一项。
19. 工作流列表支持按名称搜索，按最近更新、名称和同步状态排序。
20. 工作流可以归档和恢复，默认隐藏已归档工作流，减少误删风险。
21. 后端使用 FastAPI 提供工作流 CRUD 和模拟运行接口。
22. 后端使用 SQLite 文件 `server/data/workflow_studio.db` 存储工作流。
23. 前端可以检查后端健康状态。
24. 前端可以将当前工作流同步到后端；已有后端 ID 时使用更新，否则创建。
25. 前端可以从后端加载工作流列表，并保留浏览器本地离线能力。
26. 后端会保存工作流运行记录，包括输入、状态、步骤和创建时间。
27. 前端可以触发后端运行，并查看后端运行历史。
28. 后端提供 `POST /api/workflows/validate` 校验接口。
29. 后端会在创建、更新和运行工作流前执行结构校验，有严重问题时返回 HTTP 400。
30. 前端工作流检查优先调用后端校验接口，失败时使用本地校验兜底。
31. 后端运行历史支持按工作流筛选、删除单条和按工作流清空。
32. 前端运行历史默认加载当前工作流记录，并支持搜索、状态筛选、删除和清空。
33. 后端运行大模型节点时，如果配置 `DEEPSEEK_API_KEY` 会优先调用 DeepSeek；否则在配置 `OPENAI_API_KEY` 时调用 OpenAI；未配置或调用失败时回退模拟输出。
34. 前端运行日志会显示大模型节点来源是 `DeepSeek`、`OpenAI` 还是 `模拟输出`。
35. 前端模型状态面板会显示 DeepSeek / OpenAI 是否已配置，并提示是否会使用模拟输出。
36. 大模型真实调用失败时，后端会返回简短错误原因，前端在运行日志中单独展示。
37. 后端会读取 `server/data/knowledge/` 下的 `.md` 和 `.txt` 文档，并按关键词返回相关片段。
38. 前端知识库面板展示本地知识库文档数、片段数和目录位置。
39. 工具节点支持配置 HTTP 请求地址、方法、请求头和请求体。
40. 后端运行工具节点时会真实请求本机 HTTP 接口，并把状态码、响应内容和错误原因写入运行日志。
41. 工具节点默认只允许请求 `localhost`、`127.0.0.1` 或 `::1`。
42. 后端运行使用连线拓扑排序计算执行顺序，节点位置乱序时仍按依赖执行。
43. 条件节点支持 `sourceHandle=true` 和 `sourceHandle=false` 两种出边，运行时只执行命中的分支。
44. 运行历史列表展示状态、运行输入和输出摘要。
45. 运行结果摘要展示完成、跳过和错误数量，并支持复制整次运行结果。
46. 运行日志支持复制单个节点的输入、输出和错误原因。
47. 工作流检查中的节点级问题可以点击定位到对应节点。
48. 节点配置面板支持字段级错误/提醒展示，并可以恢复当前节点默认配置。

## 校验规则

严重问题：

- 至少需要一个用户输入节点
- 至少需要一个最终回答节点
- 工作流不能存在环形依赖
- 最终回答节点必须连接上游节点
- 输出变量名不能重复

提醒：

- 孤立节点没有任何连线
- 非用户输入节点没有上游输入

- 按最近更新、名称和同步状态排序

- 归档和恢复当前工作流
- 删除当前工作流
- 编辑工作流名称
- 自动保存当前编辑状态到浏览器
- 导入 JSON 为新的工作流
- 导出当前工作流 JSON
- 同步当前工作流到后端
- 从后端加载工作流
- 后端运行当前工作流
- 使用知识检索节点检索本地文档
- 使用工具节点调用本机 HTTP 接口
- 查看后端运行历史
- 搜索后端运行历史
- 按成功 / 失败筛选运行历史
- 复制整次运行结果
- 删除单条后端运行历史
- 清空当前工作流的后端运行历史
