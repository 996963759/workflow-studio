# 演示流程

## 目标

用 5 到 8 分钟展示项目的完整能力：账号隔离、工作流编排、后端运行、知识库、工具节点和工程化。

## 准备

```powershell
docker compose up --build
```

打开：

```text
http://127.0.0.1:8000
```

## 快速演示路线

1. 注册一个本地账号并登录。
2. 从“工作流模板”创建“客服知识库问答”。
3. 展示画布节点：用户输入、知识检索、大模型、最终回答。
4. 点击“同步到后端”，说明工作流已持久化到后端数据库。
5. 点击“同步运行”，展示逐节点输入、输出、来源、耗时和错误原因。
6. 打开“运行历史”，展示运行记录保存、搜索、筛选和删除能力。
7. 打开“版本与审计”，展示版本快照、发布状态、版本对比和审计日志。
8. 打开“系统概览”，展示数据库、队列、模型、知识库、成员和成本估算状态。
9. 创建第二个账号，说明看不到第一个账号的工作流和知识库。
10. 运行 `scripts/test-all.ps1`，说明 lint、build、unit test、smoke test。

## 进阶演示路线

### 文案生成并转语音

1. 在“管理中心 -> 模型”保存阿里云百炼 DashScope API Key。
2. 从“工作流模板”创建“文案生成并转语音”。
3. 输入一段口播需求，例如“给咖啡店新品燕麦拿铁写一段 20 秒中文口播”。
4. 点击“同步到后端”，再点击“同步运行”或“异步入队”。
5. 查看运行日志中的大模型输出、TTS provider 和音频地址。

### 营销内容一键拆分

1. 从“工作流模板”创建“营销内容一键拆分”。
2. 展示大模型先输出 JSON，再由 JSON 解析、循环和聚合节点拆出结构化内容。
3. 说明这个模板用来展示复杂工作流能力，不只是单次大模型调用。

### 团队空间和权限

1. 使用第一个账号创建邀请码。
2. 使用第二个账号通过邀请码加入团队空间。
3. 在成员管理里切换 owner/editor/viewer 角色。
4. 说明工作流、运行历史、知识库、模型配置都按团队空间隔离。

## 讲解重点

- 为什么按连线拓扑排序执行工作流。
- 本地草稿和后端持久化如何处理冲突。
- 如何用 Bearer Token 做用户隔离。
- 为什么引入 SQLAlchemy 和 Alembic。
- 知识库当前使用本地 Markdown/TXT 原文 + SQLite 哈希向量索引，检索时混合关键词和相似度。
- 为什么模型 Key 采用团队空间级配置，并且后端只返回“已配置 / 未配置”状态。
- 异步任务为什么统一走 Kafka，以及为什么测试环境临时使用 thread。
- GitHub Actions 如何自动验证前端构建和后端测试。
